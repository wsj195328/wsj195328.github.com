UISegmentedControl

　分段控制器，常用于执行多选一的情景。

接下我们看下如何用代码来设置UISegmentedControl的样式。

  1. NSArray * titleArr = @[@"新",@"研",@"科",@"技"];
  2. // 通过title数组进行初始化
  3. UISegmentedControl * seg  = [[UISegmentedControl alloc]initWithItems:titleArr];
  4. seg.frame = CGRectMake(0, 0, 300, 50);
  5. seg.center = self.view.center;
  6. //设置默认选项,代码更改也是如此
  7. seg.selectedSegmentIndex = 10;
  8. //边框颜色
  9. seg.tintColor = [UIColor yellowColor];
  10. // 7.0 后废弃的样式
  11. seg.segmentedControlStyle = UISegmentedControlStylePlain;
  12. // 设置在点击后是否弹起
  13. seg.momentary = YES;
  14. //设置指定索引的title
  15. [seg setTitle:@"❤新" forSegmentAtIndex:1];
  16. //设置指定索引的图片
  17. [seg setImage:[UIImage imageNamed:@"新研科技.png"] forSegmentAtIndex:3];
  18. // 在指定索引利用图片插入一个选项
  19. [seg insertSegmentWithImage:[UIImage imageNamed:@"mei.png"] atIndex:2 animated:NO];
  20. // 在指定索引利用title插入一个选项
  21. [seg insertSegmentWithTitle:@"insert" atIndex:3 animated:NO];
  22. // 移除指定索引的选项
  23. [seg removeSegmentAtIndex:0 animated:NO];
  24. // 设置指定某个索引选项的宽度
  25. [seg setWidth:30.f forSegmentAtIndex:2];

　由上面的UIkit继承图可以发现UISegmentedControl像UIButton一样继承于UIControl,所以它是可以用户交互的。

  1. // 对值的改变事件添加处理方法
  2. //监听的是这个事件：UIControlEventValueChanged，值改变事件
  3. [seg addTarget:seg action:@selector(segChange:) forControlEvents:UIControlEventValueChanged];
  1.  - (void)segChange:(UISegmentedControl*)seg
  2. {
  3.     // 用户当前点击的index
  4.     NSInteger index = seg.selectedSegmentIndex;
  5. }

UISlider

UISlider常用的属性表

常用属性 定义
value 当前的值
minimumValue 当值可以改变时，滑块可以滑动到最小位置的值，默认为0.0
maximumValue 当值可以改变时，滑块可以滑动到最大位置的值，默认为1.0
continuous 如果设置YES，在拖动滑块的任何时候，滑块的值都会改变。默认设置为YES
minimumValueImage 滑块条最小值处设置的图片，默认为nil
maximumValueImage 滑块条最大值处设置的图片,默认为nil
thumbTintColor 当前滑块的颜色，默认为白色
currentThumbImage 当前滑块的图片

系统不提供更改滑块大小的方法，我们可以通过设置滑块图片的大小来实现

  1. [self.slider setThumbImage:[UIImage imageNamed:@"圆勾"] forState:UIControlStateNormal];
  2. [self.slider setThumbImage:[UIImage imageNamed:@"圆勾"] forState:UIControlStateHighlighted];

UISwitch

UISwitch开关控件的常用属性如下

常用属性 定义
onTintColor 处于on时switch 的颜色
tintColor 处于off时switch 的颜色
onImage 设置on 的图标
offImage 设置off的图标
on 设置switch的开关
thumbTintColor 中间滑块的颜色

像UISlider一样，UISwitch可以增加事件响应机制。监听的是这个事件：UIControlEventValueChanged，值改变事件

  1. [self.switch addTarget:self action:@selector(switchChange:) forControlEvents:UIControlEventValueChanged];

活动指示器的常用属性和方法如下

常用属性 定义
指示器的样式 activityIndicatorViewStyle
当停止动画的时候，是否隐藏。默认为YES hidesWhenStopped

UIProgressView

进度条与UISlider不同，他不可以与用户进行交互。常用属性如下：

常用属性 定义
progressViewStyle 设置进度条风格样式
progress 设置进度条进度(0.0-1.0之间，默认为0.0)
progressTintColor 设置已走过进度的进度条颜色
trackTintColor 设置未走过进度的进度条颜色

UIStepper

这是一个用于递增的控件，继承自UIControl，它主要的事件是UIControlEventValueChanged。常用属性如下

常用属性 定义
value 当前所表示的值，默认0.0
minimumValue 最小可以表示的值，默认0.0
maximumValue 最大可以表示的值，默认100.0
stepValue 每次递增或递减的值，默认1.0
continuous 控制是否持续触发UIControlEventValueChanged事件。默认YES，即当按住时每次值改变都触发一次UIControlEventValueChanged事件，否则只有在释放按钮时触发UIControlEventValueChanged事件。
autorepeat 控制是否在按住是自动持续递增或递减。默认YES。
wraps 控制值是否在[minimumValue,maximumValue]区间内循环。默认NO。

UITextView

在之前我们学习过一个输入文本的控件UITextField,两者区别如下：

  • UITextView继承自UIScrollView,UITextField继承自UIControl。两个控件在使用上相当相似。
  • UITextView支持多行输入并且可以滚动显示浏览全文，而UITextField只能单行输入。
  • UITextView继承自UIScrollView,UITextField继承自UIView[UIControl]。
  • UITextview没有placeholder属性，UItextField有placeholder属性
