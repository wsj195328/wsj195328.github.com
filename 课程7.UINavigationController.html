1.UINavigationController介绍

　在iOS系统中UINavigationController是一个非常常用的 容器类 ViewController,常用来管理逻辑层次清晰的一组UIViewController,最经典的就是系统的设置应用。
　导航控制器的主要工作是管理视图控制器,也是提供一些自定义视图。具体地说,它提供了一个导航栏,包含一个返回按钮和一些可以自定义按钮。导航控制器还可以选择提供一个导航工具栏视图（Navigation Toolbar），我们可以将一些自定义按钮放在上面。

下图是来自官方的图解,可以看到NavigationController与它的父类有些不同，多了Navigation view 和 Navigation Toolbar。

2.UINavigationController设置

navigationItem

属于独立的UIViewController

　注意navigationItem不是UINavigationController的属性而是UIViewController的一个属性，这个属性是为UINavigationController服务的。文档中是这么解释的“The navigation item used to represent the view controller in a parent’s navigation bar. (read-only)”，即navigation item在navigation Bar代表一个viewController，具体一点儿来说就是每一个加到navigationController的viewController都会有一个对应的navigationItem，该对象由viewController以懒加载的方式创建，稍后我们可以在对象中对navigationItem进行配置。

NavigationBar

属于个NavigationController 影响所有的控制视图控制器

NavigaitonBar就是导航栏，位于屏幕的上方，管理整个NavigationController的navigationItem，即类似navigationcontroller一样提供了一个栈来管理item。

navigationItem

　如设置应用头部显示的那样，我们可以往navigationItem上边可以设置leftBarButtonItem, rightBarButtonItem, backBarButtonItem, title以及prompt等属性。前三个每一个都是一个UIBarButtonItem对象，最后两个属性是一个NSString类型描述，注意添加该描述以后NavigationBar的高度会增加30，总的高度会变成74(不管当前方向是Portrait还是Landscape，此模式下navgationbar都使用高度44加上prompt30的方式进行显示)。当然如果觉得只是设置文字的title不够爽，你还可以通过titleview属性指定一个定制的titleview，这样你就可以随心所欲了，当然注意指定的titleview的frame大小，不要显示出界。

　借住storyboard我们可以可视化的设置以上这些属性，将一个NavigationColer设置成window根视图控制器，同时将VC1 设置成Navigation的rootViewController，并设置了左右按钮和title。

UIBarButtonItem

　在上面我们可以通过storyboard拖拽的方式设置导航栏中的 leftBarButtonItem 和 rightBarButtonItem。现在我们来学习用代码如何自定义这两个 UIBarButtonItem。
查看API可以看到系统提供了四种创建的方法：

  1. // 创建系统样式的BarButtonItem
  2. - (instancetype)initWithBarButtonSystemItem:(UIBarButtonSystemItem)systemItem target:(id)target action:(SEL)action;
  4. // 根据图片创建
  5. - (instancetype)initWithImage:(UIImage *)image style:(UIBarButtonItemStyle)style target:(id)target action:(SEL)action;
  7. // 根据文字创建
  8. - (instancetype)initWithTitle:(NSString *)title style:(UIBarButtonItemStyle)style target:(id)target action:(SEL)action;
  10. // 根据传入自定义View来创建，自由度高
  11. - (instancetype)initWithCustomView:(UIView *)customView;

比如我们在VC1的viewDidLoad中设置通过文字创建左按钮

  1.  self.navigationItem.leftBarButtonItem = [[UIBarButtonItem alloc]initWithTitle:@"leftBtn" style:UIBarButtonItemStylePlain target:self action:@selector(leftBtnCLick)];

通过图片创建自定义右按钮

  1.  self.navigationItem.rightBarButtonItem = [[UIBarButtonItem alloc] initWithImage:[UIImage imageNamed:@"test"] style:UIBarButtonItemStylePlain target:self action:@selector(right:)];

自定义UIView的UIBarButtonItem

  1. UIView *testView = [[UIView alloc] initWithFrame:CGRectMake(0, 0, 40, 60)];
  2. self.navigationItem.rightBarButtonItem = [[UIBarButtonItem alloc] initWithCustomView:testView];

探索左边一共可以添加多少个按钮

3.UINavigationController的使用

　　通过storyBoard情景模拟之前三个页面跳转的例子，这次练习使用Push。


　　通过下面的运行效果图我们可以发现Push和Model不一样的跳转方式。而且左上角自带了返回按钮。

　　接下来我们看一下如何用代码实现首先我们设置一个UINavigationController当做根视图控制器，在AppDelegate的didFinishLaunchingWithOptions中

  1. //1 创建一个VC1
  2. VC1 * rootVC = [VC1 new];
  3. //2.创建一个NavigationController，把rootvc当做根视图
  4. UINavigationController * navi = [[UINavigationController alloc]initWithRootViewController:rootVC];
  5. //3.将这个NavigationController 设置成window的根视图控制器
  6. self.window.rootViewController = navi;

　　使用pushViewController: animated: 来实现页面跳转，比如从VC1跳转到VC2的代码

  1. // vc1.m 按钮点击触发事件
  2. - (void)btnCLick
  3. {
  4.     VC2 * vc = [[VC2 alloc]init];
  5.     vc.view.backgroundColor = [UIColor yellowColor];
  6.     [self.navigationController pushViewController:vc animated:YES];
  7. }

　　返回的方法有三种：

  • popViewControllerAnimated： 返回上一页
  • popToViewController：animated： 返回根视图
  • popToViewRootViewControllerAnimated　返回到某一个视图

4.UINavigationController原理

　　UINavigationController以栈的方式管理视图，各个视图的切换就是压栈和出栈操作，出栈后的视图会立即销毁。
　　UINavigationController是一个导航控制器，它用来组织有层次关系的视图，在UINavigationController中子控制器以栈的形式存储，只有在栈顶的控制器能够显示在界面中，一旦一个子控制器出栈则会被销毁。以下是官方图解
　　

　　UINavigationController默认也不会显示任何视图（这个控制器自身的UIView不会显示），它必须有一个根控制器rootViewController，而且这个根控制器不会像其他子控制器一样被销毁。
还是以系统自带的设置应用为例

5.Navigation Toolbar

　NavigationController不仅会使它所控制的UIViewController多显示一个NavigationBar(默认显示，可是设置隐藏)，它还可以显示一个Toolbar，Toolbar默认是隐藏，可以通过[self.navigationController setToolbarHidden:NO];来显示toolbar。

下面是配置toolbar的视力代码

  1. //和设置navigationBarItem类似
  2.   //先设置一个UIBarButtonItem，然后组成数组，然后把这个数组赋值给self.toolbarItems
  3.   UIBarButtonItem *btn1=[[UIBarButtonItem alloc]initWithBarButtonSystemItem:UIBarButtonSystemItemAdd target:self action:nil];
  4.   UIBarButtonItem *btn2=[[UIBarButtonItem alloc]initWithBarButtonSystemItem:UIBarButtonSystemItemBookmarks target:self action:nil];
  5.   UIBarButtonItem *btn3=[[UIBarButtonItem alloc]initWithBarButtonSystemItem:UIBarButtonSystemItemCompose target:self action:nil];
  6.   // 这个是类似弹簧效果可以撑开
  7.   UIBarButtonItem *btn4=[[UIBarButtonItem alloc]initWithBarButtonSystemItem:UIBarButtonSystemItemFlexibleSpace target:self action:nil];
  8.   NSArray *arr1=[[NSArray alloc]initWithObjects:btn4,btn1,btn4,btn2,btn4,btn3,btn4, nil];
  9.   self.toolbarItems=arr1;
